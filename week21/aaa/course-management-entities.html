מסמך ישויות וקשרים - מערכת ניהול קורס

מסמך זה מתעד את הישויות והקשרים במערכת, אחד לאחד לפי ההגדרה שסיפקת.

Users

טבלה זו מתארת משתמשי מערכת מאוחדים לצורך התחברות, זיהוי והרשאות.
סוג ישות: Strong Entity + Master Data.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של משתמש
firstName String שם פרטי
lastName String שם משפחה
email String, UNIQUE אימייל כניסה למערכת
phone String, Nullable טלפון משתמש לצרכי זיהוי/אימות
passwordHash String סיסמה מוצפנת (Hash)
twoFactorEnabled Boolean, Default false האם מוגדר לאימות דו-שלבי (2FA / MFA)
isActive Boolean האם המשתמש פעיל ויכול להתחבר
lastLoginAt DateTime, Nullable זמן התחברות אחרון
createdAt DateTime תאריך יצירת הרשומה
updatedAt DateTime תאריך עדכון אחרון

Staff

טבלה זו מתארת את אנשי הסגל במערכת והתפקיד הדומייני שלהם (מעבר לזהות ב-Users).
סוג ישות: Strong Entity (Subtype / Extension של Users).

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של איש סגל
userId FK -> Users.id, UNIQUE מזהה המשתמש המאוחד של איש הסגל לצורך התחברות והרשאות
isSuperAdmin Boolean, Default false אדמין ראשי — גישה מלאה למערכת; עוקף בדיקות Permissions הרגילות
type Enum: מפקד / מרצה / מתרגל תפקיד איש הסגל במערכת
createdAt DateTime תאריך יצירת הרשומה
updatedAt DateTime תאריך עדכון אחרון של הרשומה

Permissions

טבלה זו מתארת את ההרשאות האפשריות במערכת, לצורך ניהול גישה ויכולות.
סוג ישות: Reference Entity (Lookup Data).

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של הרשאה
key String, UNIQUE מפתח הרשאה ייחודי לשימוש לוגי במערכת
name String שם תצוגה של ההרשאה (בנוסף ל-description המפורט)
description String תיאור מורחב של ההרשאה
createdAt DateTime תאריך יצירת הרשומה

StaffPermissions

טבלה זו מתארת את השיוך בין אנשי סגל לבין הרשאות במבנה דינמי.
סוג ישות: Junction Entity.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של שיוך הרשאה לאיש סגל
staffId FK -> Staff.id איש הסגל שאליו משויכת ההרשאה
permissionId FK -> Permissions.id ההרשאה המשויכת
createdAt DateTime תאריך יצירת הרשומה
staffId + permissionId UNIQUE משולב מונע שיוך כפול של אותה הרשאה לאותו איש סגל

החלטת מודל הרשאות: מקור האמת להרשאות לסגל רגיל הוא Permissions + StaffPermissions. אין שימוש בשדה role בטבלת Users. למשתמש עם Staff.isSuperAdmin=true מבוצע עקיפה מלאה (גישה לכל הפעולות).

Cohort

טבלה זו מתארת את מחזורי הקורס ואת סטטוס המחזור לאורך הזמן.
סוג ישות: Strong Entity (Domain Entity).

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של מחזור
name String שם המחזור
startDate Date תאריך התחלת המחזור
endDate Date תאריך סיום המחזור
status Enum: PLANNED / ACTIVE / FINISHED מצב המחזור
createdAt DateTime תאריך יצירת הרשומה

Class

טבלה זו מתארת את הכיתות בקורס ואת שיוכן למחזור.
סוג ישות: Strong Entity (Domain Entity).

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של כיתה
name String שם הכיתה
cohortId FK -> Cohort.id המחזור אליו הכיתה שייכת
createdAt DateTime תאריך יצירת הרשומה

Students

טבלה זו מתארת את תלמידי הקורס ואת פרטי היסוד שלהם במערכת.
סוג ישות: Strong Entity (Subtype / Extension של Users).

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של תלמיד
userId FK -> Users.id, UNIQUE מזהה המשתמש המאוחד של התלמיד לצורך התחברות והרשאות
nationalId String, UNIQUE תעודת זהות (חובה, ייחודי במערכת)
personalNumber String, UNIQUE, Nullable מספר אישי צבאי/מערכתי (אופציונלי, ייחודי אם מוגדר)
classId FK -> Class.id הכיתה הנוכחית של התלמיד (נגזר מהשיוך הפעיל ב-StudentClass)
status Enum: ACTIVE / DROPPED / FINISHED סטטוס לימודי של התלמיד בקורס
createdAt DateTime תאריך יצירת הרשומה
updatedAt DateTime תאריך עדכון אחרון של הרשומה

הערה: סטטוס תורן נגזר מ־StudentDutyPeriods (טווח תאריכים), לא משדה בוליאני ב־Students.

StudentDutyPeriods

טבלה זו מתארת היסטוריית תקופות שבהן התלמיד משמש חניך תורן (כולל תכנון מראש).
סוג ישות: Associative / Junction Entity + Temporal Entity.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של רשומת תקופת תורנות
studentId FK -> Students.id התלמיד הנוגע לתורנות
classId FK -> Class.id, Nullable כיתה משויכת (אם התורנות קשורה לכיתה מסוימת)
fromDate Date תאריך תחילת התורנות
toDate Date, Nullable תאריך סיום (רושמים מראש גם עתיד; ריק = פעיל/ללא סיום מוגדר)
notes String, Nullable הערות (למשל: סוג תורנות, הנחיות)
createdBy FK -> Staff.id, Nullable איש הסגל שיצר/קבע את הרשומה
createdAt DateTime תאריך יצירת הרשומה
updatedAt DateTime, Nullable תאריך עדכון אחרון

חוק אינטגריטי: לכל תלמיד רצוי שלא תהיה חפיפה בין טווחי תאריכים של תקופות תורן (אכיפה ב־DB או בלוגיקה).

StudentClass

טבלה זו מתארת את היסטוריית השיוך של תלמידים לכיתות לפי טווח תאריכים.
סוג ישות: Associative / Junction Entity + Temporal Entity.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של שיוך תלמיד לכיתה
studentId FK -> Students.id התלמיד המשויך
classId FK -> Class.id הכיתה המשויכת
fromDate Date תאריך התחלת שיוך התלמיד לכיתה
toDate Date, Nullable תאריך סיום שיוך (ריק כאשר השיוך פעיל)
createdAt DateTime תאריך יצירת הרשומה
studentId + classId + fromDate UNIQUE משולב מונע פתיחה כפולה של אותו שיוך באותו תאריך התחלה

הערה: Students.classId נשמר כשיוך הכיתה הנוכחית לצרכי ביצועים ופשטות תפעול, ו-StudentClass משמש להיסטוריית מעברים בין כיתות.
חוק אינטגריטי: Students.classId חייב להיות זהה ל-classId של רשומת StudentClass הפעילה (toDate IS NULL).
חוק אינטגריטי נוסף: לכל תלמיד מותרת רשומת שיוך פעילה אחת בלבד (UNIQUE חלקי על studentId כאשר toDate IS NULL).
חוק אינטגריטי נוסף: אין חפיפה בין טווחי תאריכים של אותו תלמיד (exclusion constraint על טווחי תוקף).

StudentCohort

טבלה זו מתארת שיוך תלמיד למחזורים לאורך זמן לצרכי היסטוריה ודיווח.
סוג ישות: Associative / Junction Entity + Temporal Entity.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של שיוך תלמיד למחזור
studentId FK -> Students.id התלמיד המשויך
cohortId FK -> Cohort.id המחזור המשויך
fromDate Date תאריך התחלת שיוך למחזור
toDate Date, Nullable תאריך סיום שיוך למחזור (ריק כאשר פעיל)
createdAt DateTime תאריך יצירת הרשומה
studentId + cohortId + fromDate UNIQUE משולב מונע פתיחה כפולה של אותו שיוך לאותו תאריך התחלה

חוק אינטגריטי: לכל תלמיד מותרת רשומת מחזור פעילה אחת בלבד (UNIQUE חלקי על studentId כאשר toDate IS NULL).
חוק אינטגריטי נוסף: אין חפיפה בין טווחי מחזורים של אותו תלמיד.

StudentEvents

טבלה זו מתארת אירועים/רישומים בתיק האישי של התלמיד (יומן מנהלי-חינוכי).
סוג ישות: Strong Entity (Domain Entity) + Transactional בציר זמן.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של האירוע
studentId FK -> Students.id התלמיד אליו האירוע שייך
eventType String סוג האירוע (טקסט חופשי / תיוג לוגי שהמערכת מאפשרת)
title String כותרת האירוע
description String, Nullable תיאור מפורט
severity Enum: INFO / NOTICE / WARNING / CRITICAL, Nullable חומרה לתצוגה וסינון
visibility Enum: STAFF_ONLY / STUDENT_READ מי רשאי לראות (למשל רק סגל או גם התלמיד)
status Enum: OPEN / CLOSED / ARCHIVED מצב טיפול ברשומה
relatedClassId FK -> Class.id, Nullable כיתה קשורה לאירוע (אם רלוונטי)
occurredAt DateTime, Nullable מועד בו האירוע התרחש בעולם (אם שונה מיצירת הרשומה)
openedBy FK -> Staff.id איש הסגל שפתח/רשם את האירוע
closedBy FK -> Staff.id, Nullable איש הסגל שסגר/ארכב
closedAt DateTime, Nullable מועד סגירה
createdAt DateTime תאריך יצירת הרשומה
updatedAt DateTime תאריך עדכון אחרון

StudentEventAttachments

טבלה זו מקשרת קבצים לאירועי תלמיד.
סוג ישות: Junction Entity.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של צירוף קובץ
studentEventId FK -> StudentEvents.id האירוע המשויך
fileId FK -> Files.id הקובץ המצורף
sortOrder Number סדר תצוגה
createdAt DateTime תאריך צירוף
studentEventId + fileId UNIQUE משולב מונע צירוף כפול של אותו קובץ לאותו אירוע

StaffClass

טבלה זו מתארת את השיוך בין אנשי סגל לכיתות.
סוג ישות: Junction Entity.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של שיוך איש סגל לכיתה
staffId FK -> Staff.id איש הסגל המשויך
classId FK -> Class.id הכיתה המשויכת
staffId + classId UNIQUE משולב מונע שיוך כפול של אותו איש סגל לאותה כיתה

Files

טבלה זו מתארת קבצים שמועלים למערכת והמטא-דאטה שלהם.
סוג ישות: Asset / Resource Entity.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של קובץ
url String קישור לאחסון הקובץ
fileName String שם הקובץ
size Number גודל הקובץ בבייטים
uploadedByUserId FK -> Users.id מזהה משתמש מאוחד שהעלה את הקובץ
createdAt DateTime תאריך יצירת הרשומה

חוק אינטגריטי: הקשר למעלה קובץ נאכף דרך Users, והקישור ל-Staff/Students נגזר דרך userId.

Materials

טבלה זו מתארת את חומרי הלימוד המפורסמים לכל כיתה.
סוג ישות: Strong Entity (Domain Entity).

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של חומר לימוד
title String כותרת חומר הלימוד
description String תיאור חומר הלימוד
classId FK -> Class.id הכיתה לה מיועד החומר
createdBy FK -> Staff.id איש הסגל שיצר את חומר הלימוד
requiresSubmission Boolean האם החומר דורש הגשה מהתלמידים
dueDate DateTime, Nullable מועד אחרון להגשה (אם יש)
createdAt DateTime תאריך יצירת הרשומה

MaterialResources

טבלה זו מתארת משאבים המשויכים לחומר לימוד (קבצים, קישורים, וידאו ועוד).
סוג ישות: Dependent / Composition Entity (Child of Materials).

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של משאב חומר
materialId FK -> Materials.id החומר אליו המשאב שייך
type Enum: FILE / LINK / VIDEO סוג המשאב
fileId FK -> Files.id, Nullable קישור לקובץ פנימי כאשר הסוג הוא FILE
url String, Nullable קישור חיצוני כאשר הסוג הוא LINK/VIDEO
title String, Nullable כותרת משאב להצגה
sortOrder Number סדר הצגה בתוך החומר
createdAt DateTime תאריך יצירת הרשומה

Submissions

טבלה זו מתארת את ההגשות של תלמידים עבור חומרי לימוד.
סוג ישות: Transactional Entity.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של הגשה
studentId FK -> Students.id התלמיד שהגיש
materialId FK -> Materials.id חומר הלימוד אליו ההגשה שייכת
fileUrl String קישור לקובץ שהוגש
submissionNumber Number מספר ניסיון ההגשה עבור אותו חומר
isLatest Boolean האם זו ההגשה העדכנית ביותר עבור אותו חומר
submittedAt DateTime מועד ההגשה
status Enum: SUBMITTED / LATE סטטוס ההגשה בפועל
studentId + materialId + submissionNumber UNIQUE משולב מאפשר ריבוי הגשות ומונע כפילות ניסיון

הערה: מצב MISSING מחושב לוגית בדוחות (NOT EXISTS) ולא נשמר כרשומה פיזית.

Grades

טבלה זו מתארת ציונים לתלמידים, עבור חומר לימודי או הערכות עצמאיות.
סוג ישות: Transactional Entity + Versioned / Attempt-based Entity.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של ציון
studentId FK -> Students.id התלמיד שקיבל ציון
materialId FK -> Materials.id, Nullable חומר לימוד משויך (אם הציון קשור לחומר)
gradeType Enum: MATERIAL / EXAM / INTERVIEW / MANUAL סוג הציון שניתן
title String כותרת הציון (למשל: בוחן אמצע, הערכה ידנית)
attemptNumber Number, Default 1 מספר ניסיון עבור אותה הערכה
isFinal Boolean, Default false האם זה הציון הקובע לדיווח
score Number הציון שניתן
feedback TEXT, Nullable משוב מסגל — מומלץ לאחסן HTML (טבלאות עיצוב וכו'). ב־PostgreSQL אין טיפוס נפרד ל־HTML: משתמשים ב־TEXT או VARCHAR; התוכן הוא מחרוזת והפרדה בין "טקסט רגיל" ל־HTML היא בהסכמה באפליקציה (feedbackFormat או קונבנציית תצוגה).
feedbackFormat Enum: PLAIN / HTML, Default PLAIN האם המשוב בשדה feedback הוא טקסט רגיל או HTML (לצורך רינדור בטוח — סניטציה בצד שרת/לקוח)
gradedBy FK -> Staff.id איש הסגל שנתן את הציון
createdAt DateTime תאריך יצירת הרשומה
studentId + materialId + attemptNumber UNIQUE חלקי כאשר materialId לא ריק מאפשר היסטוריית ניסיונות ומונע כפילות ניסיון
studentId + materialId + isFinal=true UNIQUE חלקי כאשר materialId לא ריק מבטיח ציון סופי יחיד לכל תלמיד ולכל חומר
studentId + gradeType + title + isFinal=true UNIQUE חלקי כאשר materialId ריק מבטיח ציון סופי יחיד להערכה ידנית/מבחן שאינה קשורה לחומר

ClassSessions

טבלה זו מתארת מפגשי לימוד בפועל (שיעורים), כבסיס לנוכחות ומעקב יומי.
סוג ישות: Strong Entity (Operational Domain Entity).

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של מפגש
classId FK -> Class.id הכיתה של המפגש
topic String, Nullable נושא השיעור

lecturerStaffId FK -> Staff.id, Nullable איש סגל מוביל המפגש
createdAt DateTime תאריך יצירת הרשומה

Attendance

טבלה זו מתארת נתוני נוכחות תלמידים ברמת מפגש לימוד.
סוג ישות: Transactional Entity.

שם פרמטר הערות תיאור בעברית
id מזהה ראשי (PK) מזהה ייחודי של רשומת נוכחות
studentId FK -> Students.id התלמיד אליו הנוכחות שייכת

Indexing & Integrity Rules (תכנון DB)

סעיף זה מרכז כללי אינדקסים ואילוצים מומלצים לביצועים ולאמינות נתונים בסביבת ייצור.

  • אינדקס חובה על כל FK בכל הטבלאות התפעוליות.
  • אינדקסים תפעוליים מומלצים:
    • Attendance(studentId, sessionId) ו-ClassSessions(classId, startsAt)
    • AttendanceDaily(studentId, classId, date) ו-AttendanceDaily(classId, date)
    • Students(nationalId) ו-Students(personalNumber) (כאשר לא NULL)
    • StudentEvents(studentId, createdAt)
    • Submissions(studentId, materialId, isLatest) ו-Submissions(submittedAt)
    • MessageTargets(messageId, targetType) ו-MessageDeliveries(studentId, channel, status)
    • TaskAssignments(status, assignedAt) ו-Tasks(dueDate, status)
  • אילוצי CHECK מומלצים:
    • MessageTargets עבור targetType מול studentId/classId/groupId
    • FormSubmissions.submissionNumber >= 1
    • Grades.attemptNumber >= 1
    • Messages: אם type=FORM אז formId IS NOT NULL; אם type=MESSAGE אז formId IS NULL
    • StudentClass.fromDate <= toDate כאשר toDate לא ריק
    • StudentCohort.fromDate <= toDate כאשר toDate לא ריק
  • אילוצי UNIQUE חלקיים מומלצים:
    • StudentClass(studentId) כאשר toDate IS NULL
    • StudentCohort(studentId) כאשר toDate IS NULL
    • FormSubmissions(formId, studentId) כאשר isLatest=true
    • Grades(studentId, materialId) כאשר isFinal=true AND materialId IS NOT NULL
    • Grades(studentId, gradeType, title) כאשר isFinal=true AND materialId IS NULL

Soft Delete Policy (תכנון לוגי)

לשקול הוספת deletedAt לטבלאות מרכזיות לצורך שחזור, בקרה ועמידה בדרישות רגולציה:
Users, Staff, Students, Class, Materials, Forms, Messages, Tasks, Files.

סיכום קשרים (במילים פשוטות)

  • סגל <- הרשאות (דינמי)
  • מחזור -> כיתות -> תלמידים
  • תלמידים <-> מחזורים (עם היסטוריית טווח תאריכים דרך StudentCohort)
  • תלמידים <-> כיתות (עם היסטוריית טווח תאריכים דרך StudentClass)
  • כיתה -> מפגשים (ClassSessions) -> נוכחות פר־מפגש; נוכחות יומית (AttendanceDaily)
  • כיתה -> חומרים -> משאבים -> הגשות -> ציונים
  • טפסים (כולל תרחישי "סקר" בשאלה אחת) -> שאלות -> תשובות
  • הודעות -> יעדים (MessageTargets) -> משלוחים (MessageDeliveries) -> תגובות
  • תלמידים -> תקופות תורן (StudentDutyPeriods); אירועי תיק (StudentEvents) + צירופים
  • כיתות -> קבוצות -> חברי קבוצה

מקרא סוגי ישויות (מונחי DB)

Strong Entity (ישות ליבה)

ישות עסקית מרכזית שעומדת בפני עצמה ויש לה זהות עצמאית במערכת.
בדרך כלל היא שורש של תהליכים אחרים ומכילה נתוני דומיין יציבים יחסית.

Master Data

נתוני אב מערכתיים שמשמשים כבסיס רוחבי למודולים רבים.
שינויים בהם משפיעים על ישויות נוספות ולכן הם מהווים מקור אמת מרכזי.

Transactional Entity (ישות תפעולית)

ישות שמתעדת אירוע/פעולה שקרתה בזמן (מי עשה מה ומתי).
היא בדרך כלל גדלה מהר בנפח ומשמשת לדוחות, מעקב ואנליטיקה.

Junction / Associative Entity (ישות קשר)

טבלת ביניים שמקשרת בין שתי ישויות או יותר, לרוב בקשרי רבים-לרבים.
לעיתים כוללת גם שדות עסקיים נוספים (כמו סטטוס או טווח תאריכים).

Temporal Entity (ישות היסטורית-זמנית)

ישות שמנהלת היסטוריה לאורך זמן באמצעות שדות תוקף כמו fromDate/toDate.
מאפשרת לדעת "מה היה המצב בתאריך מסוים" ולא רק מה המצב הנוכחי.

Dependent / Composition Entity (ישות תלויה)

ישות שלא עומדת לבד ותלויה לוגית בישות הורה.
לרוב נמחקת/מאבדת משמעות ללא ההורה, ומשמשת לפירוק מבנה מורכב.

Reference / Lookup Entity (ישות ייחוס)

טבלת ערכי ייחוס יחסית יציבים (כמו הרשאות או קטגוריות קבועות).
נועדה למנוע hardcode ולהפוך ניהול ערכים קבועים לניתן לתחזוקה.

Asset / Resource Entity (ישות משאב)

ישות שמייצגת נכס דיגיטלי כמו קובץ, קישור או וידאו.
משמשת לשיוך משאבים לתכנים שונים תוך שמירה על מטא-דאטה אחיד.

Audit Entity (ישות ביקורת)

ישות append-only שמטרתה תיעוד שינויים לצורך בקרה, מעקב ותחקור.
בדרך כלל לא מעדכנים רשומות קיימות אלא רק מוסיפים אירועים חדשים.

Communication Aggregate Root (שורש אגרגט תקשורת)

ישות מרכזית שמנהלת תהליך תקשורת ומאגדת תחתיה הודעות, יעדים ומסירות.
מגדירה גבולות עקביות לדומיין התקשורת ומרכזת את הלוגיקה העסקית סביבו.
